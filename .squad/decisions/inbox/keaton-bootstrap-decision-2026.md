### 2026-03-05: Bootstrap removal decision
**By:** Keaton (Lead)
**What:** Keep Bootstrap 4.4.1 (CSS and JS) for this redesign iteration. Do not remove it now.
**Why:** The Bootstrap grid (`col-xl-8 offset-xl-2`, `col-lg-10 offset-lg-1`, `row`, `container-md`) is load-bearing structural layout in `post.html` — removing it mid-redesign forces a template rewrite that isn't scoped to this iteration. Marcus asked for aesthetic improvements; Fenster and McManus can override Bootstrap CSS cleanly without fighting it. The `data-toggle="tooltip"` on pagination links also activates Bootstrap's JS plugin, so the full jQuery + Popper + Bootstrap JS chain is technically in use.
**Impact:** Bootstrap stays for this iteration. However, the custom JS (`scroll-to-top.js`, `search.js`) is 100% vanilla — neither file touches jQuery. That means jQuery, Popper.js, and Bootstrap JS are loaded solely to power the pagination tooltips. Dropping those tooltips and swapping to native `title` attributes (or a CSS-only approach) would let us cut three CDN requests (jQuery slim 87KB, Popper 21KB, Bootstrap JS 60KB) with minimal template work. Flag as a Phase 2 quick-win performance item for McManus once the aesthetic pass is done.
